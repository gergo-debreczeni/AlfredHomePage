Alfredbot

Your loyal ChatOps butler.

Installation

To install Alfredbot you can either visit https://slack.com/apps/A2ED8G9J8-alfredbot or use the Add to Slack button. Once Alfredbot has been added to your team you can deploy the cloudformation stack for setting up roles. Alfredbot does NOT and never will store any information about your users. The only reason it needs read access to user profile and channel/groups/usergroups information is to set up role mappings.

Configuration

Once you have configured the roles you have to add channel/group/usergroup mapping. Usage:/alfred-admin Detailed command options: /alfred-admin add Example:/alfred-admin add channel general arn:aws:iam:::role/readonly 2 /alfred-admin remove <type> <name> Example:/alfred-admin remove channel general` /alfred-admin list

The priority in the add command is needed so that in cases when the user invoking an aws command belongs to multiple channels that have different roles mapped the most open one would be used. Only a slack administrator can use /alfred-admin command to set up mappings. Let's assume he adds the following configuration: /alfred-admin add usergroup admins arn:aws:iam:::role/FullAdmin 1 /alfred-admin add channel ops arn:aws:iam:::role/PowerUser 2 /alfred-admin add channel general arn:aws:iam:::role/ReadOnly 3 If Bob belongs to the admins usergroup and is on general channel we want him to have the most open role, hence the one with priority 1 is chosen.

After the slack administrator configured the roles use /alfred-sync start to sync the user permissions with alfredbot.

Usage

After the role mappings have been configured any user can simply invoke /alfred-invoke
